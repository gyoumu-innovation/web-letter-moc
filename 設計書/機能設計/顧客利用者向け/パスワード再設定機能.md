## ステップ1：再設定リクエスト（password-reset-request.html）

### 入力内容

| 項目の論理名 | 項目の物理名 | 入力形式 | 必須 |
|---|---|---|---|
| メールアドレス | email | テキストボックス | ○ |

### 出力内容

| ステータス | 内容 | 備考 |
|---|---|---|
| 成功 / 該当なし | 「メールを送信しました」画面へ遷移 | 該当なしでも同じ画面へ（列挙攻撃対策） |

### 処理フロー

```
1. メールアドレスを入力して「送信」ボタンをクリック
2. バリデーション
   ├─ 未入力 → MSG-003
   ├─ メール形式不正 → MSG-004
   └─ エラーなし → 続行
3. usersテーブルでメールアドレス検索
   ├─ 該当なし → セキュリティのため成功と同じ画面へ遷移（エラー表示しない）
   └─ 該当あり → 続行
4. 再設定トークン生成（ランダム文字列・有効期限60分）
5. トークンをDBに保存
6. 再設定メール送信（IF-01）
7. 「メールを送信しました」画面へ遷移
```

### バリデーション

| チェック内容 | メッセージID | 表示タイミング |
|---|---|---|
| email 未入力 | MSG-003 | フォーカスアウト / 送信時 |
| email 形式不正 | MSG-004 | フォーカスアウト / 送信時 |

## ステップ2：パスワード再設定（password-reset.html）

### 入力内容

| 項目の論理名 | 項目の物理名 | 入力形式 | 必須 |
|---|---|---|---|
| 新パスワード | new_password | パスワードボックス | ○ |
| 新パスワード（確認） | new_password_confirm | パスワードボックス | ○ |

### 出力内容

| ステータス | 内容 | 備考 |
|---|---|---|
| 成功 | ログイン画面へ遷移 | 完了メッセージを表示 |
| トークン無効 | エラー画面表示 | MSG-009 を表示 |

### 処理フロー

```
1. メール内のURLをクリック
2. トークンの有効性チェック
   ├─ 無効・期限切れ → MSG-009 を表示して終了
   └─ 有効 → パスワード再設定フォーム表示
3. 新パスワード・確認パスワードを入力して「設定」ボタンをクリック
4. バリデーション
5. Have I Been Pwned で照合
   ├─ 流出済み → MSG-008 を表示
   └─ 問題なし → 続行
6. パスワードをbcrypt（コストファクター12）でハッシュ化して更新
7. トークン無効化
8. ログイン画面へ遷移（完了メッセージ表示）
```

### バリデーション

| チェック内容 | メッセージID | 表示タイミング |
|---|---|---|
| 新パスワード 未入力 | MSG-005 | フォーカスアウト / 送信時 |
| 12文字未満 | MSG-006 | フォーカスアウト / 送信時 |
| 128文字超過 | MSG-007 | フォーカスアウト / 送信時 |
| 流出パスワード | MSG-008 | 送信時 |
| 確認パスワード 不一致 | MSG-010 | フォーカスアウト / 送信時 |
| トークン無効・期限切れ | MSG-009 | ページ読み込み時 |
