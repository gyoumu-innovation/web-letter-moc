# 機能設計書：パスワード再設定機能

## 概要

| 項目 | 内容 |
|---|---|
| 機能ID | F-U-002 |
| 機能名 | パスワード再設定機能 |
| 対象ユーザー | 顧客利用者 |
| 対応画面 | パスワード再設定リクエスト画面 / メール送信完了画面 / パスワード再設定画面 |

---

## 機能説明

ログインパスワードを忘れたユーザーが、メールアドレスを入力して再設定用URLを受け取り、新しいパスワードを設定する。

---

## 処理フロー

### ステップ1：再設定リクエスト（password-reset-request.html）

```
1. ユーザーがメールアドレスを入力して「送信」ボタンをクリック
2. 入力バリデーション（メール形式チェック）
3. usersテーブルでメールアドレス検索
   ├─ 該当なし → セキュリティのため「送信しました」と表示（エラーは出さない）
   └─ 該当あり → 続行
4. 再設定トークン生成（ランダム文字列、有効期限60分）
5. トークンをDBに保存
6. 再設定メール送信（IF-01）
7. 「メールを送信しました」画面へ遷移（password-reset-sent.html）
```

### ステップ2：パスワード再設定（password-reset.html）

```
1. ユーザーがメール内のURLをクリック
2. トークンの有効性チェック
   ├─ 無効・期限切れ → エラー画面表示
   └─ 有効 → パスワード再設定フォーム表示
3. ユーザーが新パスワード・確認パスワードを入力して「設定」ボタンをクリック
4. 入力バリデーション
5. パスワードをハッシュ化してusersテーブル更新
6. トークン無効化
7. ログイン画面へ遷移（完了メッセージ表示）
```

---

## 画面項目

### 再設定リクエスト画面

| 項目名 | 入力種別 | 必須 | バリデーション |
|---|---|---|---|
| メールアドレス | テキスト | ○ | メール形式、254文字以内 |

### パスワード再設定画面

| 項目名 | 入力種別 | 必須 | バリデーション |
|---|---|---|---|
| 新パスワード | パスワード | ○ | 8文字以上、英数字記号含む |
| 新パスワード（確認） | パスワード | ○ | 新パスワードと一致 |

---

## エラーメッセージ

| 条件 | メッセージ |
|---|---|
| メール形式不正 | メールアドレスの形式が正しくありません |
| トークン無効・期限切れ | このリンクは無効または期限切れです。再度リクエストしてください |
| パスワード不一致 | パスワードが一致しません |
| パスワード強度不足 | パスワードは8文字以上で英数字を含めてください |

---

## 関連テーブル

| テーブル | 操作 | 条件 |
|---|---|---|
| users | SELECT | email = 入力値 |
| password_reset_tokens | INSERT | トークン保存 |
| password_reset_tokens | SELECT | token = URLのトークン |
| users | UPDATE | password_hash 更新 |
| password_reset_tokens | DELETE / UPDATE | トークン無効化 |

※ password_reset_tokens テーブルは別途定義要
