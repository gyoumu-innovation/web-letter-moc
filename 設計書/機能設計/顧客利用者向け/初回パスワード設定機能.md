# 機能設計書：初回パスワード設定機能

## 概要

| 項目 | 内容 |
|---|---|
| 機能ID | F-U-025 |
| 機能名 | 初回パスワード設定機能 |
| 対象ユーザー | 顧客利用者（招待済み・未ログイン） |
| 対応画面 | 初回パスワード設定画面（password-init.html） |

契約者専用システム。社内担当者がアカウントを作成した際に招待メールが自動送信され、ユーザーがリンクから初めてパスワードを設定する。

---

## トリガー

社内担当者がユーザーアカウントを新規作成したとき（システム自動送信）

---

## ステップ1：招待メール送信（社内担当者側）

```
1. 社内担当者がユーザーアカウントを新規作成
2. システムが初回パスワード設定用トークンを生成
3. 招待メール（R-06）を送信
```

---

## ステップ2：初回パスワード設定（顧客利用者側）

### 入力内容

| 項目の論理名 | 項目の物理名 | 入力形式 | 必須 |
|---|---|---|---|
| 新しいパスワード | new_password | パスワードボックス | ○ |
| 新しいパスワード（確認） | new_password_confirm | パスワードボックス | ○ |

### 出力内容

| ステータス | 内容 | 備考 |
|---|---|---|
| 成功 | ログイン画面へ遷移 | 完了メッセージを表示 |
| トークン無効 | エラー表示 | MSG-009 を表示 |

### 処理フロー

```
1. 招待メール内のリンクをクリック
2. トークンの有効性チェック
   ├─ 無効・期限切れ → MSG-009 を表示して終了
   └─ 有効 → 初回パスワード設定フォームを表示
3. 新パスワード・確認パスワードを入力して「設定する」クリック
4. バリデーション
5. Have I Been Pwned で照合
   ├─ 流出済み → MSG-008 を表示
   └─ 問題なし → 続行
6. パスワードをbcrypt（コストファクター12）でハッシュ化して保存
7. トークン無効化
8. ログイン画面へ遷移（完了メッセージ表示）
```

### バリデーション

| チェック内容 | メッセージID | 表示タイミング |
|---|---|---|
| 新パスワード 未入力 | MSG-005 | フォーカスアウト / 送信時 |
| 12文字未満 | MSG-006 | フォーカスアウト / 送信時 |
| 128文字超過 | MSG-007 | フォーカスアウト / 送信時 |
| 流出パスワード | MSG-008 | 送信時 |
| 確認パスワード 不一致 | MSG-010 | フォーカスアウト / 送信時 |
| トークン無効・期限切れ | MSG-009 | ページ読み込み時 |

文言は `メッセージ定義書.md` 参照。

## トークン仕様

| 項目 | 内容 |
|---|---|
| 有効期限 | **未定（要決定）** ※ パスワード再設定（60分）より長めが望ましい（例：24時間・72時間） |
| 使用回数 | 1回のみ有効（設定完了後に無効化） |

## パスワード再設定機能との違い

| 項目 | 初回パスワード設定 | パスワード再設定 |
|---|---|---|
| トリガー | 社内担当者がアカウント作成 | ユーザー自身がリクエスト |
| メールテンプレート | R-06（招待メール） | R-05（再設定メール） |
| トークン有効期限 | 長め（24〜72時間） | 60分 |
| 対象 | 未ログインユーザー | 既存ユーザー |
