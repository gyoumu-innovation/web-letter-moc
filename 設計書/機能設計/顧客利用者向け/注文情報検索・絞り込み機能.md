# 機能設計書：注文情報検索・絞り込み機能

## 概要

| 項目 | 内容 |
|---|---|
| 機能ID | F-U-012 |
| 機能名 | 注文情報検索・絞り込み機能 |
| 対象ユーザー | 顧客利用者 |
| 対応画面 | 注文一覧画面（order-list.html） |

---

## 機能説明

注文一覧のタブ・日付・注文番号による絞り込み検索。  
注文一覧・参照機能（F-U-006）のサブ機能として動作する。

## 検索条件

| 条件 | 種別 | 処理方式 |
|---|---|---|
| ステータスタブ | 完全一致 | クライアントサイドフィルタ |
| ご注文日（From〜To） | 範囲検索 | クライアントサイドフィルタ |
| 発送予定日（From〜To） | 範囲検索 | クライアントサイドフィルタ |
| 注文番号 | 部分一致 | クライアントサイドフィルタ |

## 処理フロー

```
1. 画面初期表示時に全注文を取得・表示
2. 条件変更時（タブクリック or 検索ボタン）
3. 取得済みデータをJSでフィルタリング
4. 件数カウントを更新
5. 0件の場合は「該当する注文が見つかりませんでした」を表示
```

## 備考

- 初期表示時のみDBアクセス（全件取得）
- 件数が多い場合は初期表示をページネーション化することを検討
