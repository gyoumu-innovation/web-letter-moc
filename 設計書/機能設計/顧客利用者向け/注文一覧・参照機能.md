# 機能設計書：注文一覧・参照機能

## 概要

| 項目 | 内容 |
|---|---|
| 機能ID | F-U-006 |
| 機能名 | 注文一覧・参照機能 |
| 対象ユーザー | 顧客利用者 |
| 対応画面 | 注文一覧画面（order-list.html） |

---

## 機能説明

ログイン中のユーザーが所属する企業の注文を一覧で確認する。  
ステータスタブ・日付・注文番号で絞り込みが可能。

---

## 画面項目

### 表示項目（注文カード）

| 項目名 | 表示内容 |
|---|---|
| ステータスバッジ | 下書き / 未確認 / 確認待ち / 差し戻し / 生産中 / 発送済み |
| 注文名 | 商品種別＋通数（例：封筒＋便箋×333通） |
| 注文番号 | orders.order_no |
| 注文日 | orders.order_date |
| 発送予定日 / 発送日 | ステータスに応じて切り替え |

### 絞り込み項目

| 項目名 | 入力種別 | 説明 |
|---|---|---|
| ステータスタブ | タブ切替 | 全て / 下書き / 未確認 / 確認待ち / 生産中 / 発送済み |
| ご注文日（From〜To） | 日付 | 注文日の範囲絞り込み |
| 発送予定日（From〜To） | 日付 | 発送予定日の範囲絞り込み |
| 注文番号 | テキスト | 部分一致検索 |

---

## 処理フロー

```
1. 画面初期表示
   - ログインユーザーの company_id に紐づく注文を全件取得
   - 注文日の降順で表示
   - 件数を表示

2. ステータスタブ切替
   - 選択したステータスで再絞り込み（クライアントサイドで処理）

3. 絞り込み「検索する」ボタンクリック
   - 入力された条件で再絞り込み（クライアントサイドで処理）

4. カードの「確認する」ボタンクリック
   - 注文詳細画面へ遷移

5. 下書きカードの「編集する」ボタンクリック（PCのみ）
   - 注文内容編集画面へ遷移

6. 下書きカードの「削除する」ボタンクリック
   - 削除確認ダイアログ表示
   - 確定 → orders.status を削除 or 論理削除
```

---

## ステータス表示ルール

| ステータス値 | 表示名 | バッジ色 | 表示ボタン |
|---|---|---|---|
| draft | 下書き | グレー | 編集する（PCのみ）・削除する |
| new | 未確認 | 赤 | 確認する |
| confirmed | 確認待ち | 青 | 確認する |
| returning | 差し戻し | 赤 | 確認する |
| producing | 生産中 | 緑 | 確認する |
| shipped | 発送済み | 紺 | 確認する |

## 備考

- スマホ表示では「編集する」ボタンは非表示（入稿フォームはPC専用）
- 絞り込みはクライアントサイドで処理（初期表示時に全件取得）
- 件数が多い場合はページネーションを検討
