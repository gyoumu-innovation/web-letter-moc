# 機能設計書：注文内容編集機能

## 概要

| 項目 | 内容 |
|---|---|
| 機能ID | F-U-008b |
| 機能名 | 注文内容編集機能 |
| 対象ユーザー | 顧客利用者（PCのみ） |
| 対応画面 | 新規注文作成画面（入稿作成_統合.html）※既存注文読み込み |

---

## 機能説明

下書き保存した注文、または差し戻された注文を入稿フォームで再編集する。

## 編集可能条件

| ステータス | 編集可否 |
|---|---|
| draft（下書き） | ○ |
| returning（差し戻し） | ○ |
| new / confirmed / producing / shipped | × |

## 処理フロー

```
1. 注文一覧の「編集する」ボタン（下書き）または
   注文詳細の「編集する」ボタン（差し戻し）をクリック
2. order_id をパラメータとして入稿フォームへ遷移
3. 既存データをフォームに反映して表示
4. ユーザーが内容を修正して「注文する」または「下書き保存」
5. 注文機能（F-U-009）と同じ保存処理へ
```

## 関連テーブル

注文機能（F-U-009）と同様。  
既存データをSELECTしてフォームに初期値としてセット。
