# 機能設計書：入稿内容保存機能

## 概要

| 項目 | 内容 |
|---|---|
| 機能ID | F-U-010 |
| 機能名 | 入稿内容保存機能（下書き保存） |
| 対象ユーザー | 顧客利用者 |
| 対応画面 | 新規注文作成画面（各ステップ） |

---

## 機能説明

入稿フォームの途中で「下書き保存」ボタンを押した際に、入力途中の内容をdraftステータスで保存する。

## 処理フロー

```
1. ユーザーが「下書き保存」ボタンをクリック
2. 現在ステップまでの入力データをバリデーション（必須チェックは緩い）
3. ordersテーブルに status=draft で INSERT or UPDATE
4. 各仕様テーブルに保存（未入力項目はNULL）
5. 「下書き保存しました」のトースト通知を表示
6. 画面はそのまま（遷移しない）
```

## 保存タイミング

- 「下書き保存」ボタンの手動クリック
- ブラウザを閉じる前の確認ダイアログ（実装依存）
