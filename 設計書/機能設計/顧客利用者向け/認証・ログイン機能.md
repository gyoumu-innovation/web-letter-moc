# 機能設計書：認証・ログイン機能

## 概要

| 項目 | 内容 |
|---|---|
| 機能ID | F-U-001 |
| 機能名 | 認証・ログイン機能 |
| 対象ユーザー | 顧客利用者 |
| 対応画面 | ログイン画面（index.html） |

---

## 機能説明

顧客利用者がメールアドレスとパスワードを入力し、システムにログインする。  
認証成功後はセッションを発行し、ログイン後トップ画面（新規注文作成）へ遷移する。

---

## 画面項目

| 項目名 | 入力種別 | 必須 | バリデーション |
|---|---|---|---|
| メールアドレス | テキスト | ○ | メール形式、254文字以内 |
| パスワード | パスワード | ○ | 1文字以上 |
| ログイン状態を保持する | チェックボックス | - | - |

---

## 処理フロー

```
1. ユーザーがメールアドレス・パスワードを入力して「ログイン」ボタンをクリック
2. 入力バリデーション
   ├─ エラーあり → エラーメッセージ表示（画面遷移なし）
   └─ エラーなし → 続行
3. usersテーブルでメールアドレス検索
   ├─ 該当なし → 「メールアドレスまたはパスワードが正しくありません」
   └─ 該当あり → 続行
4. パスワードハッシュ照合
   ├─ 不一致 → 「メールアドレスまたはパスワードが正しくありません」
   └─ 一致 → 続行
5. is_initial_password = true の場合
   └─ 初回パスワード設定画面へ強制遷移
6. セッション発行
7. 「ログイン状態を保持する」がONの場合 → RememberMeトークン発行
8. ログイン後トップ画面へ遷移
   - PC：新規注文作成画面
   - スマホ（UA判定）：注文一覧画面
```

---

## エラーメッセージ

| 条件 | メッセージ |
|---|---|
| 必須未入力 | 入力してください |
| メール形式不正 | メールアドレスの形式が正しくありません |
| 認証失敗 | メールアドレスまたはパスワードが正しくありません |

---

## セキュリティ要件

- 連続ログイン失敗時のアカウントロック（失敗N回で一定時間ロック）
- パスワードはbcryptハッシュ化
- セッションID推測困難な乱数生成
