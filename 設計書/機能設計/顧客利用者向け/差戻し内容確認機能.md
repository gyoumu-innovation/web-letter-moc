# 機能設計書：差戻し内容確認機能

## 概要

| 項目 | 内容 |
|---|---|
| 機能ID | F-U-008 |
| 機能名 | 差戻し内容確認機能 |
| 対象ユーザー | 顧客利用者 |
| 対応画面 | 注文詳細画面（ステータス：差し戻し） |

---

## 機能説明

社内担当者から差し戻された注文について、差し戻し理由を確認する。  
注文詳細画面内で差し戻し理由パネルを表示する（注文ステータス確認機能の差し戻し状態）。

---

## 表示項目

| 項目名 | 表示内容 |
|---|---|
| 差し戻し理由 | return_reasons.reason（最新1件） |
| 差し戻し日時 | return_reasons.created_at |

---

## 処理フロー

```
1. 通知メール（IF-02）内のリンク or 注文一覧から注文詳細へ遷移
2. ステータスが「差し戻し」の場合
   → 差し戻し理由パネルを表示
3. ユーザーが差し戻し理由を確認
4. 「編集する」ボタンをクリック（PCのみ）
   → 注文内容編集画面へ遷移
```

---

## 表示制御

| 条件 | 表示 |
|---|---|
| ステータス = 差し戻し | 差し戻し理由パネル 表示 |
| ステータス ≠ 差し戻し | 差し戻し理由パネル 非表示 |
| スマホ表示 | 「編集する」ボタン 非表示 |
| PC表示 | 「編集する」ボタン 表示 |

## 外部IF

- IF-02（通知メール）からの遷移を想定
