# 機能設計書：パスワード変更機能

## 概要

| 項目 | 内容 |
|---|---|
| 機能ID | F-U-003 |
| 機能名 | パスワード変更機能 |
| 対象ユーザー | 顧客利用者（ログイン済み） |
| 対応画面 | パスワード変更画面（password-change.html） |

---

## 処理フロー

```
1. ヘッダーのユーザーメニュー「パスワードの変更」から遷移
2. 現在のパスワード・新パスワード・確認パスワードを入力して「変更する」クリック
3. バリデーション（必須・形式・一致チェック）
4. 現在のパスワードを照合
   ├─ 不一致 → 「現在のパスワードが正しくありません」
   └─ 一致 → 続行
5. 新パスワードをハッシュ化してusersテーブル更新
6. 完了メッセージ表示 → 元の画面へ戻る
```

## 画面項目

| 項目名 | 必須 | バリデーション |
|---|---|---|
| 現在のパスワード | ○ | 1文字以上 |
| 新しいパスワード | ○ | 8文字以上、英数字含む |
| 新しいパスワード（確認） | ○ | 新パスワードと一致 |
