# 機能設計書：入力バリデーション機能（共通）

## 概要

| 項目 | 内容 |
|---|---|
| 機能ID | F-C-001〜003 |
| 機能名 | 入力必須項目チェック / 入力形式チェック / 業務ルールチェック |
| 対象ユーザー | 顧客利用者・社内担当者（全ユーザー） |
| 対応画面 | 入力フォームを持つ全画面 |

---

## 機能説明

フォーム送信時に入力値の妥当性を検証する共通機能。  
フロントエンド（即時フィードバック）とバックエンド（二重チェック）の両方で実施する。

---

## バリデーション種別

### F-C-001：入力必須項目チェック

| チェック内容 | エラーメッセージ |
|---|---|
| 必須項目が空の場合 | {項目名}を入力してください |
| 必須選択項目が未選択の場合 | {項目名}を選択してください |
| ファイル必須なのに未選択の場合 | ファイルを選択してください |

---

### F-C-002：入力形式チェック

| 項目 | チェック内容 | エラーメッセージ |
|---|---|---|
| メールアドレス | RFC5321準拠の形式、254文字以内 | メールアドレスの形式が正しくありません |
| パスワード | 8文字以上、英字・数字を含む | パスワードは8文字以上で英数字を含めてください |
| 郵便番号 | 数字7桁（ハイフン任意） | 郵便番号は7桁の数字で入力してください |
| 電話番号 | 数字10〜11桁（ハイフン任意） | 電話番号の形式が正しくありません |
| 日付 | YYYY-MM-DD形式、実在する日付 | 日付の形式が正しくありません |
| 文字数上限 | 各項目の最大文字数を超えた場合 | {N}文字以内で入力してください |
| ファイル形式 | 許可されたMIMEタイプ以外 | {許可形式}のファイルを選択してください |
| ファイルサイズ | 最大サイズ超過 | ファイルサイズは{N}MB以内にしてください |

---

### F-C-003：業務ルールチェック

| チェック内容 | エラーメッセージ |
|---|---|
| メールアドレス重複（ユーザー登録時） | このメールアドレスはすでに登録されています |
| 残り通数不足（注文時） | 残り通数が不足しています（残り{N}通） |
| 契約プラン有効期限切れ（注文時） | 契約プランの有効期限が切れています |
| パスワード確認一致チェック | パスワードが一致しません |
| 発送予定日が過去日（注文時） | 発送予定日は本日以降の日付を入力してください |
| 削除不可データの削除（使用中の差出人情報等） | この{対象}は注文で使用されているため削除できません |

---

## 実装方針

| タイミング | 実装場所 | 目的 |
|---|---|---|
| 入力中・フォーカスアウト | フロントエンド（JS） | 即時フィードバック |
| フォーム送信時 | フロントエンド（JS） | 送信前の確認 |
| APIリクエスト受信時 | バックエンド | 改ざん・二重送信防止 |

---

## 共通エラー表示ルール

- エラーは該当項目の直下に赤字で表示
- フォーム送信時は最初のエラー項目へスクロール
- 複数エラーは全件表示する
