# エラーハンドリング設計書

## 概要

| 項目 | 内容 |
|---|---|
| ドキュメントID | F-C-010 |
| 対象 | ロボットレターWEB注文システム 全画面・全API |
| 対象ユーザー | 顧客利用者・社内担当者（全ユーザー） |

本書はシステム全体のエラーハンドリング方針を定義する。  
入力バリデーションの詳細は `入力バリデーション機能.md`（F-C-001〜003）を参照。

---

## エラーの分類

| 分類 | 説明 | 例 |
|---|---|---|
| **バリデーションエラー** | ユーザーの入力値が不正 | 必須項目未入力、形式不正 |
| **認証エラー** | ログイン状態・権限の問題 | 未ログイン、トークン期限切れ、権限不足 |
| **業務エラー** | ビジネスルール違反 | 残通数不足、使用中データの削除 |
| **外部サービスエラー** | 外部システムの障害 | メール送信失敗、ストレージアクセス失敗 |
| **ネットワークエラー** | 通信の問題 | タイムアウト、接続断 |
| **システムエラー** | サーバー内部の障害 | DB接続エラー、予期しない例外 |

---

## エラーコード定義

### 認証・認可系

| エラーコード | HTTPステータス | 意味 | ユーザー向けメッセージ |
|---|---|---|---|
| `AUTH_REQUIRED` | 401 | 未ログイン・トークンなし | ログインしてください |
| `TOKEN_EXPIRED` | 401 | JWTトークン期限切れ | セッションが切れました。再度ログインしてください |
| `TOKEN_INVALID` | 401 | トークン不正 | ログインしてください |
| `FORBIDDEN` | 403 | 操作権限なし | この操作は許可されていません |
| `INVALID_CREDENTIALS` | 401 | ログイン情報不一致 | メールアドレスまたはパスワードが正しくありません |

### バリデーション系

| エラーコード | HTTPステータス | 意味 | ユーザー向けメッセージ |
|---|---|---|---|
| `VALIDATION_ERROR` | 400 | 入力値バリデーション失敗 | 入力内容に誤りがあります |
| `REQUIRED_FIELD` | 400 | 必須項目未入力 | {項目名}を入力してください |
| `INVALID_FORMAT` | 400 | 形式不正 | {項目名}の形式が正しくありません |
| `EXCEEDED_MAX_LENGTH` | 400 | 文字数超過 | {項目名}は{N}文字以内で入力してください |
| `INVALID_FILE_TYPE` | 400 | 許可外のファイル形式 | {許可形式}のファイルを選択してください |
| `EXCEEDED_FILE_SIZE` | 400 | ファイルサイズ超過 | ファイルサイズは{N}MB以内にしてください |

### 業務ルール系

| エラーコード | HTTPステータス | 意味 | ユーザー向けメッセージ |
|---|---|---|---|
| `DUPLICATE_EMAIL` | 409 | メールアドレス重複 | このメールアドレスはすでに登録されています |
| `INSUFFICIENT_QUOTA` | 422 | 残通数不足 | 残り通数が不足しています（残り{N}通） |
| `CONTRACT_EXPIRED` | 422 | 契約プラン期限切れ | 契約プランの有効期限が切れています |
| `RESOURCE_IN_USE` | 422 | 使用中リソースの削除不可 | この{対象}は注文で使用されているため削除できません |
| `INVALID_STATUS_TRANSITION` | 422 | 不正なステータス遷移 | この操作は現在のステータスでは実行できません |
| `PAST_DATE` | 422 | 過去日の指定 | 発送予定日は本日以降の日付を入力してください |

### リソース系

| エラーコード | HTTPステータス | 意味 | ユーザー向けメッセージ |
|---|---|---|---|
| `NOT_FOUND` | 404 | リソースが存在しない | 指定のデータが見つかりません |
| `ALREADY_DELETED` | 404 | 論理削除済みリソース | 指定のデータは既に削除されています |

### システム系

| エラーコード | HTTPステータス | 意味 | ユーザー向けメッセージ |
|---|---|---|---|
| `INTERNAL_ERROR` | 500 | サーバー内部エラー | システムエラーが発生しました。時間をおいて再度お試しください |
| `SERVICE_UNAVAILABLE` | 503 | 外部サービス障害 | 現在サービスが一時停止しています。時間をおいて再度お試しください |
| `TIMEOUT` | 504 | タイムアウト | リクエストがタイムアウトしました。再度お試しください |

## バックエンドのエラーレスポンス形式

内部IF設計書で定義したレスポンス形式に従う。

### 単一エラー
```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "入力内容に誤りがあります",
    "details": null
  }
}
```

### フィールド単位のバリデーションエラー
```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "入力内容に誤りがあります",
    "details": {
      "fields": [
        { "field": "email", "code": "INVALID_FORMAT", "message": "メールアドレスの形式が正しくありません" },
        { "field": "name", "code": "REQUIRED_FIELD", "message": "氏名を入力してください" }
      ]
    }
  }
}
```

### 業務エラー（パラメータ付き）
```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "INSUFFICIENT_QUOTA",
    "message": "残り通数が不足しています（残り50通）",
    "details": {
      "remaining": 50,
      "requested": 100
    }
  }
}
```

## フロントエンドのエラーハンドリング方針

### API呼び出し共通処理

```
APIレスポンス受信
  ├─ success: true → 正常処理
  └─ success: false → エラーコードに応じて処理
       ├─ 401 (AUTH_REQUIRED / TOKEN_EXPIRED) → ログイン画面へリダイレクト
       ├─ 403 (FORBIDDEN) → アクセス拒否メッセージ表示
       ├─ 400 (VALIDATION_ERROR) → フィールドエラーをフォームに表示
       ├─ 404 (NOT_FOUND) → 「データが見つかりません」表示
       ├─ 409 / 422 (業務エラー) → エラーメッセージをトースト表示
       ├─ 500 (INTERNAL_ERROR) → システムエラーダイアログ表示
       └─ ネットワーク断 → 「通信エラー」トースト表示・リトライボタン
```

### エラー表示パターン

| パターン | 適用シーン | 表示形式 |
|---|---|---|
| **インラインエラー** | フォーム入力バリデーション | 項目直下に赤字テキスト |
| **トースト通知** | API呼び出し結果・業務エラー | 画面右上に3秒表示 |
| **モーダルダイアログ** | システムエラー・重要な警告 | 操作を止めて表示 |
| **ページ内バナー** | セッション切れ・権限エラー | ページ上部に固定表示 |
| **エラーページ** | 404・403 | 専用ページへ遷移 |

### トースト通知の仕様

| 種別 | 色 | 表示時間 | 例 |
|---|---|---|---|
| 成功 | 緑 | 3秒 | 保存しました |
| エラー | 赤 | 5秒（または手動クローズ） | メールアドレスが重複しています |
| 警告 | 黄 | 4秒 | 変更が保存されていません |
| 情報 | 青 | 3秒 | ファイルをアップロードしています |

## シナリオ別ハンドリング

### セッション期限切れ

```
1. APIから401（TOKEN_EXPIRED）が返る
2. 現在のページURLをsessionStorageに保存
3. 「セッションが切れました。再度ログインしてください」を表示
4. 3秒後にログイン画面へリダイレクト
5. ログイン成功後、保存したURLへ戻る
```

### ネットワーク断・タイムアウト

```
1. fetchがrejectまたはタイムアウト（30秒）
2. トースト：「通信エラーが発生しました。再度お試しください」を表示
3. 操作中のフォームデータは保持する（ページリロード不要）
4. 下書き保存中だった場合は「自動保存に失敗しました」を表示
```

### ファイルアップロード失敗

```
1. アップロードAPI（IF-04/IF-05）がエラー
   ├─ INVALID_FILE_TYPE → 「{形式}のファイルを選択してください」
   ├─ EXCEEDED_FILE_SIZE → 「ファイルサイズは{N}MB以内にしてください」
   ├─ バックエンドバリデーション失敗 → エラー内容を詳細表示
   └─ ストレージエラー → 「アップロードに失敗しました。再度お試しください」
2. ファイル選択をリセットして再選択できる状態にする
```

### 注文確定時の業務エラー

```
1. 注文確定APIがエラー
   ├─ INSUFFICIENT_QUOTA → 残通数・必要通数をモーダルで表示
   ├─ CONTRACT_EXPIRED → 契約期限切れをモーダルで表示（担当者へ連絡を促す）
   └─ INVALID_STATUS_TRANSITION → 「この操作は現在実行できません。画面を更新してください」
```

### 差戻し・ステータス更新時の競合

```
1. 社内担当者がステータス更新APIを呼び出す
2. 409 または INVALID_STATUS_TRANSITION が返る場合
3. 「他の担当者が既に操作しています。画面を更新してください」を表示
4. 画面を最新データに更新する
```

## バックエンドのエラーログ設計

### ログレベル

| レベル | 適用ケース |
|---|---|
| `ERROR` | システムエラー（500系）・外部サービス障害 |
| `WARN` | 業務エラー（400系）・認証失敗の連続発生 |
| `INFO` | 通常のAPI呼び出し・ステータス変更 |
| `DEBUG` | 開発時の詳細トレース（本番では無効） |

### ログ出力項目

```json
{
  "timestamp": "2026-05-21T10:30:00Z",
  "level": "ERROR",
  "request_id": "uuid-xxxx",
  "user_id": 123,
  "method": "POST",
  "path": "/api/orders/45/submit",
  "status_code": 500,
  "error_code": "INTERNAL_ERROR",
  "message": "DB connection failed",
  "stack_trace": "...(本番では非表示)"
}
```

### セキュリティ上の注意

- パスワード・トークンはログに出力しない
- スタックトレースはAPIレスポンスには含めない（ログにのみ記録）
- 認証失敗が短時間に5回以上続く場合は `WARN` レベルでアラート対象とする

## エラーページ仕様

| ページ | 表示条件 | 表示内容 |
|---|---|---|
| 404ページ | 存在しないURLへアクセス | 「ページが見つかりません」＋注文一覧へのリンク |
| 403ページ | 権限のないページへアクセス | 「アクセスする権限がありません」＋注文一覧へのリンク |
| 500ページ | サーバーエラーが発生 | 「システムエラーが発生しました」＋管理者連絡先 |
