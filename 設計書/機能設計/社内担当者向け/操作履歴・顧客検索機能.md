## F-S-005：操作履歴参照

### 機能説明
社内担当者の操作ログを一覧表示する。不正操作の追跡や監査目的で使用する。

### 表示項目

| 項目 | 表示内容 |
|---|---|
| 日時 | operation_logs.created_at |
| 操作者 | operation_logs.staff_id → staff.name |
| 操作種別 | operation_logs.action（ステータス変更・差戻し・ユーザー追加等） |
| 対象 | operation_logs.target_type + target_id（例：orders/123） |
| 詳細 | operation_logs.detail（変更前後の値等） |

### 絞り込み項目

| 項目 | 入力形式 |
|---|---|
| 操作者 | セレクトボックス（staff一覧） |
| 操作種別 | セレクトボックス |
| 日付範囲（From〜To） | 日付ピッカー |

### 処理フロー

```
1. 操作履歴画面へアクセス（adminロール以上）
2. operation_logsをSELECT（JOIN staff）
3. 絞り込み条件があればWHERE句に追加
4. 日時の降順で表示
```

### アクセス制御

- adminロールのみアクセス可能
- staffロールは自身の操作履歴のみ参照可能

## F-S-006：顧客情報検索・絞り込み

### 機能説明
登録済み顧客企業・ユーザーを検索・一覧表示する。

### 表示項目（企業一覧）

| 項目 | 表示内容 |
|---|---|
| 企業ID | companies.id |
| 企業名 | companies.name |
| 契約プラン | plans.name |
| 契約開始日 | companies.contract_start_date |
| ユーザー数 | users COUNT |
| 当月注文数 | orders COUNT（当月） |

### 絞り込み項目

| 項目 | 入力形式 |
|---|---|
| 企業名 | テキスト（部分一致） |
| 契約プラン | セレクトボックス |
| 契約開始日（From〜To） | 日付ピッカー |

### 処理フロー

```
1. 顧客一覧画面へアクセス（staffロール以上）
2. companies + plans JOIN でSELECT
3. usersとordersはサブクエリでCOUNT
4. 絞り込み条件があればWHERE句に追加
5. 企業名の昇順で表示
```

### 企業詳細

企業カードをクリックすると、その企業のユーザー一覧・注文一覧を表示する。
