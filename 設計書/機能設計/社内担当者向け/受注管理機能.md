## 概要

| 機能ID | 機能名 | 画面 |
|---|---|---|
| F-S-001 | 受注一覧・参照機能 | admin-order-list.html |
| F-S-002 | 受注内容確認機能 | admin-order-detail.html |
| F-S-003 | 受注情報検索・絞り込み機能 | admin-order-list.html |
| F-S-004 | 差戻し機能 | admin-order-detail.html |

---

## F-S-001：受注一覧・参照

### 機能説明
全顧客企業の受注を一覧表示する。ステータスごとにタブ切り替えが可能。

### 表示項目

| 項目 | 表示内容 |
|---|---|
| 受注ID | orders.id |
| 企業名 | companies.name |
| ステータス | orders.status（日本語変換） |
| 注文日時 | orders.created_at |
| 送付通数 | orders.recipient_count |
| 担当者 | orders.assigned_staff_id → staff.name |

### ステータスタブ

| タブ | 対象ステータス |
|---|---|
| 全件 | 全ステータス |
| 新規受注 | new |
| 確認中 | confirmed |
| 制作中 | producing |
| 差し戻し | returned |
| 発送済み | shipped |

### ソート

デフォルト：注文日時の降順。カラムクリックで昇順/降順切替。

## F-S-002：受注内容確認

### 機能説明
受注の全詳細情報を確認する。顧客が入稿した内容（封筒仕様・便箋仕様・文面・送付リスト）を社内担当者が確認する。

### 表示項目

| セクション | 表示内容 |
|---|---|
| 基本情報 | 受注ID・企業名・注文日・ステータス |
| 封筒仕様 | order_envelope_specs（サイズ・窓・印刷方法等） |
| 便箋仕様 | order_stationery_specs（フォント・書き方・枚数等） |
| 文面 | order_bodies（文面テキスト・差出人情報） |
| 同封物 | order_enclosures（同封物種別・枚数） |
| 送付リスト | mailing_lists（ファイルリンク・件数） |

### 操作ボタン

| ボタン | 動作 |
|---|---|
| ステータス変更 | ドロップダウンでステータスを選択・更新 |
| 差し戻し | 差戻しモーダルを表示（F-S-004） |
| 送付リストDL | mailing_listsのファイルをDL |

---

## F-S-003：受注情報検索・絞り込み

### 絞り込み項目

| 項目 | 入力形式 |
|---|---|
| 企業名 | テキスト（部分一致） |
| 受注ID | テキスト（完全一致） |
| ステータス | セレクトボックス |
| 注文日（From〜To） | 日付ピッカー |
| 担当者 | セレクトボックス（staff一覧） |

### 処理フロー

```
1. 絞り込み条件を入力
2. 「検索」ボタンクリック
3. WHERE句に条件を追加してSELECT
4. 結果を一覧に再表示
```

## F-S-004：差戻し機能

### 機能説明
受注内容に問題がある場合、差戻し理由を記録してステータスを「returned」に変更する。  
顧客には通知（IF-02）が送られる。

### 処理フロー

```
1. 受注詳細画面「差し戻し」ボタンクリック
2. 差戻しモーダル表示
   - 差戻し理由（テキストエリア、必須）
   - 確認チェックボックス
3. 「差し戻し確定」クリック
4. return_reasonsテーブルにINSERT
5. orders.statusを'returned'に更新
6. IF-02で顧客へ通知メール送信
7. notificationsテーブルにINSERT
8. 一覧に戻る
```

### バリデーション

| チェック | エラーメッセージ |
|---|---|
| 差戻し理由が空 | 差戻し理由を入力してください |
| 確認チェックなし | 確認チェックを入れてください |
