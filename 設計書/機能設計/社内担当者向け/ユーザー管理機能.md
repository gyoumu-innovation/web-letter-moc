## F-S-007：ユーザー追加

### 機能説明
顧客企業のユーザーアカウントを社内担当者が新規作成する。

### 入力項目

| 項目名 | 必須 | 説明 |
|---|---|---|
| 企業 | ○ | companiesから選択（セレクトボックス） |
| 氏名 | ○ | 100文字以内 |
| メールアドレス | ○ | ユニーク制約あり |
| 初期パスワード | ○ | 8文字以上、英数字混在 |
| ロール | ○ | rolesから選択 |

### 処理フロー

```
1. ユーザー一覧「追加」→ 登録フォーム表示
2. 入力・バリデーション
3. usersテーブルにINSERT（パスワードはbcryptハッシュ化）
4. 初期パスワード通知メール送信（IF-01相当）
5. 一覧に戻る
```

### バリデーション

| チェック | エラーメッセージ |
|---|---|
| メールアドレス重複 | このメールアドレスは既に使用されています |
| パスワード形式不正 | 8文字以上で英数字を含めてください |
| 企業未選択 | 企業を選択してください |

## F-S-008：ユーザー編集・削除

### 編集
登録と同じフォームに既存値を初期表示して更新。  
パスワード欄は空欄の場合は変更しない。

### 削除

```
1. ユーザー一覧「削除する」クリック → 確認モーダル表示
2. 「削除する」確定
3. 進行中の注文が存在しないか確認
   ├─ あり → エラーメッセージ（削除不可）
   └─ なし → 論理削除（deleted_atにタイムスタンプ）
4. 一覧に反映
```

### バリデーション（削除時）

| チェック | エラーメッセージ |
|---|---|
| 進行中注文あり | 進行中の注文があるため削除できません |

## F-S-009：ロール管理

### 機能説明
ユーザーのロールをユーザー一覧または編集画面から変更する。

### ロール一覧

| ロール | 権限範囲 |
|---|---|
| customer_admin | 企業の全注文参照・注文作成・ユーザー管理（社内非表示） |
| customer_user | 自分の注文参照・注文作成 |
| staff | 全受注参照・ステータス変更・差戻し |
| admin | 全機能（ユーザー管理含む） |

### 処理フロー

```
1. ユーザー編集画面でロールを変更
2. users.role_idをUPDATE
3. 操作ログをoperation_logsに記録
```
