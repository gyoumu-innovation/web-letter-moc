## 週別スケジュール

| 週 | Phase | 主要タスク |
|---|---|---|
| W1 | 設計 | 未設計機能の設計書作成・DB設計書更新（権限テーブル追加） |
| W2 | 設計 | 技術スタック決定・インフラ構成確定・API設計書作成 |
| W3 | 環境構築 | Docker環境・ステージング環境構築・CI/CD・Datadog設定・SES申請 |
| W4 | 開発 | 認証・ログイン・RBACの基盤実装 |
| W5 | 開発 | 権限制御完了・入稿機能開発 |
| W6 | 開発 | 受注管理・ステータス管理・差戻し機能 |
| W7 | 開発 | ユーザー管理・企業管理 |
| W8 | 開発 | 通知・メール送信・外部IF（ろぼすけ）連携 |
| W9 | 開発 | 代理ログイン・操作履歴・残機能・コードレビュー |
| W10 | テスト | 単体テスト・結合テスト実施 |
| W11 | テスト | システムテスト・不具合修正 |
| W12 | テスト / リリース | UAT・本番環境構築・リリース準備・リリース判定 |
| W13 | リリース | 本番リリース・初動確認・監視強化期間 |

|---|---|

---

## バッファ考慮

| リスク発生時 | 対応 |
|---|---|
| W9時点で機能未完了 | 低優先機能（操作履歴・代理ログイン）をv1.1に先送り |
| W11時点でテスト不合格多数 | W12を不具合修正週に充当・リリースをW13→W14に延期 |
| SES未承認 | SendGridで代替し、SES承認後に切り替え |
