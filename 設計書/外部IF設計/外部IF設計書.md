## IF詳細

### IF-01：メール送信（パスワード再設定）

| 項目 | 内容 |
|---|---|
| **連携先** | メールサーバー（SMTP） |
| **方向** | システム → メールサーバー |
| **トリガー** | ユーザーがパスワード再設定リクエストを送信したとき |
| **プロトコル** | SMTP / TLS |

**送信メール仕様：**

| 項目 | 内容 |
|---|---|
| 宛先 | リクエストされたメールアドレス |
| 件名 | 【ロボットレター】パスワード再設定のご案内 |
| 本文 | 再設定URL（有効期限付きトークン含む） |
| トークン有効期限 | 60分 |

**エラー処理：**
- メールアドレスが未登録の場合でも、セキュリティ上「送信しました」と表示する
- 送信失敗時はリトライ処理を行う（最大3回）

---

### IF-02：メール送信（通知）

| 項目 | 内容 |
|---|---|
| **連携先** | メールサーバー（SMTP） |
| **方向** | システム → メールサーバー |
| **トリガー** | 注文ステータス変更時（差し戻し・発送済み） |
| **プロトコル** | SMTP / TLS |

**通知種別と件名：**

| 通知種別 | 件名 |
|---|---|
| 注文が差し戻された | 【ロボットレター】注文が差し戻されました（注文番号：{order_no}） |
| 注文が発送された | 【ロボットレター】注文が発送されました（注文番号：{order_no}） |
| 注文が確認待ちになった | 【ロボットレター】注文内容をご確認ください（注文番号：{order_no}） |

**エラー処理：**
- 送信失敗時はリトライ処理（最大3回）
- 失敗した場合はnotificationsテーブルの `sent_at` をNULLのまま保持

### IF-03：送付リストテンプレートダウンロード

| 項目 | 内容 |
|---|---|
| **連携先** | ファイルストレージ（S3 or 相当） |
| **方向** | ストレージ → システム → ユーザーブラウザ |
| **トリガー** | ユーザーがテンプレートダウンロードボタンをクリック |

**ファイル仕様：**

| 項目 | 内容 |
|---|---|
| ファイル形式 | Excel（.xlsx） |
| ファイル名 | mailing_list_template.xlsx |
| 保存場所 | ストレージ固定パス（バージョン管理） |
| 更新者 | 社内担当者がサンプル文面と同様にアップロードで更新 |

---

### IF-04：送付リストアップロード

| 項目 | 内容 |
|---|---|
| **連携先** | ファイルストレージ（S3 or 相当） |
| **方向** | ユーザーブラウザ → システム → ストレージ |
| **トリガー** | ユーザーがファイルを選択してアップロード |

**入力ファイル仕様：**

| 項目 | 内容 |
|---|---|
| ファイル形式 | Excel（.xlsx） / CSV（.csv） |
| 最大ファイルサイズ | 10MB |
| 必須カラム | 氏名、郵便番号、都道府県、住所1 |

**処理：**
1. ファイルをストレージに保存
2. バリデーション（形式・必須カラム・文字コード）
3. 件数をmailing_listsテーブルに記録
4. バリデーションエラーがある場合は保存せずエラー内容を返す

**エラー処理：**
- 形式不正：「ファイル形式が正しくありません（.xlsx または .csv）」
- サイズ超過：「ファイルサイズは10MB以内にしてください」
- 必須カラム不足：「{カラム名}が含まれていません」

---

### IF-05：サンプル文面アップロード

| 項目 | 内容 |
|---|---|
| **連携先** | ファイルストレージ（S3 or 相当） |
| **方向** | 社内担当者ブラウザ → システム → ストレージ |
| **トリガー** | 社内担当者がサンプル文面ファイルをアップロード |
| **利用者** | 社内担当者のみ |

**入力ファイル仕様：**

| 項目 | 内容 |
|---|---|
| ファイル形式 | PDF / 画像（.jpg, .png） |
| 最大ファイルサイズ | 20MB |
| 公開範囲 | 全顧客利用者が閲覧可能 |

**処理：**
1. ファイルをストレージに保存
2. 公開URLを発行
3. 顧客の「サンプル文面確認機能」から参照可能にする

---

## 共通事項

### ファイルストレージ

- 保存パス命名規則：`{環境}/{リソース種別}/{ID}/{ファイル名}`
  - 例：`prod/mailing_lists/1234/list_20260120.xlsx`
- アクセス制御：署名付きURL（有効期限あり）でダウンロード
- バックアップ：ストレージの冗長化設定に従う

### メール送信共通

- 送信元アドレス：`noreply@robot-letter.jp`（仮）
- HTMLメール / テキストメール両方対応
- 送信ログはnotificationsテーブルで管理
