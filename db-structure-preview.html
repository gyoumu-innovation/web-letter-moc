商品マスタ テーブル構造案

得意先ごとに商品・価格を管理できる構成

マスタ（商品定義）
m_product_categories
商品カテゴリマスタ
カラム名型説明
id PK INT —
code VARCHAR(50) envelope / stationery / option / enclosure
name VARCHAR(100) 封筒・便箋・オプション…
display_order INT 表示順
m_products
商品マスタ
カラム名型説明
id PK INT —
category_id FK INT → m_product_categories
name VARCHAR(100) 大礼紙・角2封筒…
base_price INT デフォルト加算価格（円）
condition_note VARCHAR(200) 「角2封筒のみ」などの注記 NULL可
is_active BOOLEAN 有効フラグ
display_order INT 表示順
得意先別設定（上書き）
company_products
得意先別商品設定
カラム名型説明
id PK INT —
company_id FK INT → companies
product_id FK INT → m_products
is_available BOOLEAN この得意先で表示するか
custom_price INT 価格上書き NULL=デフォルト継承
UNIQUE — UQ (company_id, product_id)
継承ルール
① company_products にレコードなし
→ 全商品デフォルト公開
② is_available = FALSE
→ この得意先では非表示
③ custom_price に値あり
→ base_price を上書き
④ custom_price = NULL
→ base_price をそのまま使用
注文時スナップショット（既存テーブル・変更なし）
order_envelope_specs
注文時封筒仕様（スナップショット）
カラム名型説明
id PK INT —
order_id FK INT → orders
product_id FK INT → m_products（参照のみ）
product_name VARCHAR(100) 注文時点の名称コピー
price_at_order INT 注文時点の価格コピー
なぜスナップショットが必要か
商品マスタを後から変更しても、
過去の注文内容・価格は変わらないようにするため。

m_products の値は「今の定義」、
order_*_specs の値は「その時の事実」。

商品一覧を組み立てるクエリイメージ
-- 得意先A向けの有効な商品一覧を取得
SELECT p.id, p.name,
COALESCE(cp.custom_price, p.base_price) AS price,
COALESCE(cp.is_available, TRUE) AS is_available
FROM m_products p
LEFT JOIN company_products cp
ON cp.product_id = p.id
AND cp.company_id = :company_id
WHERE p.is_active = TRUE
AND COALESCE(cp.is_available, TRUE) = TRUE
ORDER BY p.display_order
